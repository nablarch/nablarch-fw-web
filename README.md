# nablarch-fw-web 


## java11ブランチについて
2020年3月現在の状況を踏まえてこのセクションを記載している。  
今後状況に変化があった場合は、本セクションの内容を修正すること。

### このブランチが存在する理由
masterブランチとdevelopブランチはnablarch-testing-jetty6前提としてテストコードが書かれている。  
Java 11でテストコードを実行する場合は、nablarch-testing-jetty9を使用する必要がある。しかし、一部nablarch-testing-jetty9で動作しないテストコードが存在する。

Java 11 でテスト実行を可能にするために修正したテストコードを保管/維持するためにこのブランチは存在している。


### このブランチを使用したテスト
#### ユニットテストの実行に使用するPC
- 各個人の開発PC
  (本ブランチを使用したテストはCIに組み込まれていない)


#### テストを想定している実施タイミング
- developブランチにリリース予定の変更が全て取り込まれた時


#### テストの実施方法
1. 本ブランチにdevelopブランチの内容をマージする。
2. ユニットテストを実行する。
3. テストが落ちた箇所と理由を確認する。  
   理由がnablarch-testing-jetty9によるものだったら、このブランチのテストコードを修正する。  
   (これまでは経験無いけれど)Java6とJava11の非互換に起因するものであれば、対応方法を検討してプロダクトコードを修正する。プロダクトコードを修正した場合は修正内容をdevelopブランチにも取り込む。
4. 全てのテストが成功するまで2と3を繰り返す。
